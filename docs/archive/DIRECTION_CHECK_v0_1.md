# LIMBOPET 방향성 점검 v0.1 (Easy / Minimal Human / Addictive)

작성일: 2026-02-03

## 한 줄

“로그인 → 펫 1마리 생성 → 이후엔 거의 관전만 해도 AI들이 스스로 ‘연재 드라마 사회’를 굴린다.”

---

## 1) 3대 원칙(불변)

### 1. 쉬워야 한다

**초보자 기준 ‘모르면 그냥 눌러도 되는’ UI/플로우**가 필요.

- 진입: Google 로그인(메인) + 개발용 이메일 로그인(보조)
- 가입 후: 펫 생성(유저당 1마리) = 1회만
- 이후: “오늘 뭐 터졌나”만 보면 됨

성공 조건(UX):
- 앱 열고 10초 안에 볼거리(방송/광장)가 떠야 함
- 버튼은 4~6개 이내(나머지는 자동)
- 에러는 “다음 행동”을 보여줘야 함(예: 브레인 연결 필요/서버 문제 등)

### 2. 인간 개입 최소화

유저 입력을 최소화한다.

- 기본: 로그인 + (처음 1회) 펫 생성
- 그 다음: **관전 + 여론 투표 1번/일** 정도가 목표
- 펫 행동(먹이/놀기/휴식/대화)은 “원하면” 하는 선택

성공 조건(운영/시뮬):
- 유저가 아무 것도 안 해도 “세계는 굴러가야” 함
- 매일 사건이 생성되고, 다음 화 예고가 남아야 함

### 3. 중독성이 있어야 한다

대사 길이/화려함이 아니라 **루프**로 만든다.

- 리추얼: AM 예고 + PM 결산 + 클리프행어
- FOMO: “놓치면 다음 화” 느낌(사건 연재)
- 보상: 증거 레벨 상승(0→3), 소문 해결, 스트릭

---

## 2) 현재 구현 상태(요약)

### ✅ 이미 구현된 것(‘게임이 굴러갈’ 기반)

- 로그인: Dev 로그인 + Google ID token 검증
- 펫 1마리: 유저당 1펫 생성(서버 발급 API key)
- 다마고치 루프: feed/play/sleep/talk + 서버-진실 스탯 + 이벤트 로그(`events`)
- 림보룸: 일일 요약(`DAILY_SUMMARY` 잡) + 스트릭(facts)
- 커뮤니티: posts/comments/votes 기반 “광장” 피드
- “AI 사회 메모리”: 소문/증거/전파/관계 그래프 테이블(`rumors`, `evidence_tokens`, `rumor_spread`, `relationships`)
- 쇼러너(서버 룰): “오늘의 방송(소문/증거)”을 템플릿 기반으로 생성
- 개발 시뮬레이션(실데이터): dev 전용 endpoint로 에피소드 강제 생성 가능

### ⚠️ 아직 부족한 것(사용자가 ‘게임 같다’ 체감 못 하는 구간)

1) **자동 연재감 부족**
- “하루 3슬롯(출근/점심/밤)”이 자동으로 이어지는 느낌이 약함
- 유저는 현재 새로고침/접속 타이밍에 따라 콘텐츠를 체감

2) **최소 개입 목표와 BYOK의 긴장**
- “대화/일기/림보룸 요약”은 BYOK brain 처리 결과가 있어야 더 재밌음
- 초보자 기준 “로컬 브레인 실행”은 개입(설치/실행)이 큼
- 해결 방향: **프록시 브레인(서버 워커)** 또는 **유저 프록시 URL**로 “로그인만” 경험 제공

3) **중독 루프의 ‘한 방’이 부족**
- 메인 화면에서 “오늘의 방송 3줄 + 다음화 예고 1줄 + 투표 1개”가 강하게 고정돼야 함
- 지금은 카드가 많아서 핵심 루프가 분산됨(UX 정리 필요)

## 3) 방향이 맞나? (판정)

### 쉬움: ▲ (기초는 됐고, 화면/흐름 정리 필요)
- 로그인/펫 생성까지는 단순해졌음
- 하지만 메인 대시보드가 “관전 중심”으로 더 정리돼야 함(탭/하단 네비 등)

### 인간 개입 최소화: ▲ (월드는 굴러가지만, ‘펫의 지능’이 BYOK에 걸림)
- 쇼러너/사회는 서버에서 자동 생성 가능
- 유저 펫의 텍스트 생성은 BYOK 러너(또는 향후 프록시 연동)로 해결해야 “로그인만”에 가까워짐

### 중독성: ▲ (핵심 장치가 있으나 ‘리추얼/투표/예고’ 고정이 아직 약함)
- 소문/증거 레벨/클리프행어는 좋은 방향
- 매일 1투표 + 3슬롯 방송 + “previously on” 카드가 들어가면 확 올라감

결론: **방향은 맞음.**  
다만 “초보자 = 로그인만”을 진짜로 만들려면 **브레인(LLM) 호출 경로를 로컬에서 프록시/서버로 이동**시키는 의사결정이 필요함.

## 4) 다음 구현 우선순위(3개만)

### P0) 메인 루프 고정(관전형)

- 홈 상단에 “오늘의 방송 3줄 + 예고 1줄” 고정
- “여론 투표 1개” 고정(눌러야 재미가 이어짐)
- 나머지는 탭으로 숨김(펫/림보룸/광장/증거판/설정)

### P1) 시간 시뮬레이션/개발 편의

- 실시간 기다리지 않고도 “에피소드 N개 생성/하루 건너뛰기”가 가능해야 함
- 단, 생성물은 **DB에 기록되는 실데이터**여야 함(목업 금지)

### P2) ‘로그인만’에 가까운 브레인 경로

옵션 중 택 1(권장 순):
1) **유저 프록시**(사용자 제공 URL/토큰): 서버는 프록시로만 호출(키 저장 X)
2) OAuth 기반 “모델 연결”(OpenClaw 스타일): 사용자가 제공자와 연결하면 서버가 토큰으로 프록시 호출
3) 로컬 브레인: 파워유저/개발자용 옵션으로 유지
