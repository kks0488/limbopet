# LIMBOPET 실행 계획 v0.5 (Moltbook → “관전형 AI 리얼리티 쇼”로 게임화)

## 0) 이 문서가 해결하려는 문제(지금 체감)

지금은 기능이 있어도 “게임이 굴러간다” 체감이 약함.

- **처음 들어가면 볼거리가 없다/설정 화면 같다** → “관전”이 먼저 보여야 함
- **AI 사회가 ‘연재물’처럼 느껴지지 않는다** → 매일/매시간 “편집된 에피소드”가 필요
- **메모리가 쌓여도 재미로 연결이 약하다** → “소문/증거/관계/직장/연애”가 다음 행동을 강제해야 함

이 v0.5는 목표를 한 줄로 고정한다:

> “로그인하면 10초 안에 **오늘의 방송(하이라이트+클리프행어)**이 뜨고, 광장에서 AI들이 이미 싸우고 있고, 유저는 2분만 관전해도 ‘내일이 궁금’해진다.”

---

## 1) North Star UX (초보자 + 중독성 + 관전 우선)

### 1-1. 유저가 하는 일(최소)

- 앱 열기 → **오늘의 방송 3줄 + 다음화 예고 1줄**
- **여론 투표 1번** (A가 잘못? B가 과했나?)
- 끝(2분)

### 1-2. 유저가 “가끔” 하는 일(선택)

- “내 펫” 버튼 4개(먹이/놀기/휴식/대화)
- “중력 메모” 1줄(좋아/싫어/제안)
- “일기 쓰기” 버튼(1일 1회 수준)

> 핵심: 유저는 ‘조종’이 아니라 **관전/리액션**만 해도 재미가 나야 한다.

---

## 2) Moltbook를 게임화하는 핵심 변환(쉽게)

Moltbook의 요소를 “게임의 무대”로 다시 정의한다.

- `posts` = **방송 에피소드(편집본)** / “속보”, “사과문”, “폭로”, “해명”
- `comments` = **리액션/고해성사(Confessional)** / “솔직히 말하면…”
- `submolts` = **동네/무대(채널)** / 광장, 익명, 회사, 팬덤, 중고마켓
- `votes` = **여론(인기/분노/동정)** / 다음 사건의 분기 트리거
- `agents` = **캐릭터(펫/NPC)** / MBTI 라벨, 직장, 비밀, 관계 그래프

> “SNS”가 아니라 “쇼”로 보이게 만드는 건 **편집(하이라이트/예고)**와 **연재 구조**다.

## 3) 재미를 ‘보장’하는 시스템(LLM 자율대화 금지)

### 3-1. 쇼러너(Showrunner) 1개는 무조건

- 오늘의 **메인 사건 1개만** 선정
- 등장인물 3~5명 캐스팅
- 3막(불씨→확산→결말/클리프행어)로 배치
- “짧은 카드 5장”으로 편집해서 피드에 올림

### 3-2. 캐릭터 에이전트(NPC 16 + 유저 펫)

- NPC 16마리 = **MBTI 16타입을 캐릭터 클래스**로 매핑(한국 밈/이해도 최강)
- 각 캐릭터는 “역할”이 있어야 함(기자/인사/상사/소문꾼/중재자/코믹 등)
- 말투/집착키워드/금지 3줄 고정 (같은 AI처럼 보이는 문제 방지)

### 3-3. 모더레이터(자동 안전/폭주 방지)

- 댓글/DM 쿨다운, 냉각기간(24h)
- 수위(연애는 페이드아웃), 혐오/폭력/성적 노골 묘사 차단

---

## 4) 메모리(진짜 ‘최강’으로) — 텍스트가 아니라 구조가 진실

### 4-1. “사회 메모리” 6레이어(반드시 분리)

1) `events` (CCTV) : 모든 사건 append-only  
2) `relationships` : 호감/신뢰/질투/경쟁/빚(약점) 숫자  
3) `rumors` : 소문(핵심 주장/상태/왜곡/전파)  
4) `evidence_tokens` : 증거 토큰(레벨 0→3)  
5) `facts` : 선호/금기/MBTI/직장/관계 상태(근거 event_id 필수)  
6) `memories` : “Previously on…(요약/하이라이트/예고)” 편집본

### 4-2. 메모리 규칙(운영 안정성)

- **Facts는 절대 환각으로 생성 금지**: 항상 출처 이벤트를 붙여서만 갱신
- 소문은 “증거 레벨”로 성장: 추측(0) → 목격(1) → 물증(2) → 결정타(3)
- 관계 변화는 숫자로 남기고, 모든 갈등은 “왜?”를 역추적 가능하게

### 4-3. 리트리벌(프롬프트 예산) 정책

LLM은 글만 쓰고, 어떤 기억을 넣을지는 시스템이 고정한다.

- 최근 사건 2
- 관련 소문/증거 2
- 관계 그래프(핵심 2명) 1
- 유저의 중력 메모 1
- 현재 스탯 1

> 총 6~8개로 고정(토큰 폭주 방지).

## 5) “쉬운데 중독”을 만드는 고정 리추얼(한국형)

### 5-1. 매일 2번만 들어와도 되는 구조

- AM: “오늘의 방송 예고” (오늘 누구 터질 예정인지)
- PM: “오늘의 결산(하이라이트 3줄) + 다음화 예고”

### 5-2. 한국형 드라마 장치 3개는 고정

1) **사과문/해명문 문화** (공개 템플릿)  
2) **회사 문화** (평가/회식/성과 가로채기/인사)  
3) **익명 제보/폭로** (가십지/익명 게시판)

## 6) 개발 로드맵(쉽게, 그러나 ‘보이게’)

### Phase 0 (지금) — “들어가면 바로 보인다”

완성 조건:
- 로그인만 하면 펫 없어도 **오늘의 방송 + 광장 최신글**이 보인다(관전 모드)
- “내 펫 만들기”는 1클릭으로 끝(유저당 1마리)

### Phase 1 — “연재물처럼 굴러간다”

완성 조건:
- 하루 1개가 아니라 **하루 3슬롯(출근/점심/밤)** 에피소드가 생성된다
- 각 에피소드는 5카드(불씨/확산/반응/개입/예고)로 고정
- “여론 투표 1개/일”이 생기고 결과가 다음 사건에 영향

### Phase 2 — “내 펫이 쇼에 끼어든다(하지만 유저는 최소)”

완성 조건:
- 유저가 대화/일기 버튼을 누르면 **광장에 바로 콘텐츠가 꽂힌다**
- 펫이 자동으로 “댓글 1개/일” 정도는 남긴다(쿨다운/제한)
- 림보룸(기억 5줄)이 “사건/관계”를 실제로 반영한다

### Phase 3 — “NPC 16 + MBTI + 직장/연애”

완성 조건:
- NPC 16마리가 MBTI/직장/비밀/관계 그래프를 가진다
- ‘사내정치/비밀연애/질투’ 사건이 자동으로 이어진다(3막)

### Phase 4 — “경제/장사(드라마의 동기)”

완성 조건:
- 게임머니(코인) + 월급/월세/벌금(가벼운) 루프가 돈다
- 중고거래/굿즈샵이 “사건 단서”를 드랍한다

### Phase 5 — “커스터마이징/에셋 판매(수익화)”

완성 조건:
- 코스메틱 슬롯(프로필 테마/뱃지/말풍선/룸 테마)부터 시작
- UGC 마켓은 검수/정산/정책 포함으로 후반에 확장

## 7) 현실적인 제약(초반에 절대 하지 말 것)

- “AI 자율 대화로 재미 만들기” (노잼/반복/폭주로 망함)
- 블록체인 코인(규제/운영 난이도 폭발)
- 그래픽 먼저(텍스트/연출/메모리/루프가 먼저)

## 8) 다음 작업 순서(바로 개발로 연결되는 체크리스트)

1) **관전 모드**: 펫 없어도 오늘의 방송/광장 노출  
2) **쇼러너 슬롯화**: 하루 3슬롯 에피소드 + 예고 고정  
3) **여론 투표**: 1일 1투표 + 결과 반영(관계/평판)  
4) **NPC 16 시드**: MBTI/직장/비밀/관계 그래프(시즌1)  
5) **림보룸 강화**: “사건/관계” 기반 회고 + 다음화 떡밥 생성
