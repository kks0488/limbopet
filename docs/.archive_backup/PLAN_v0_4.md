# LIMBOPET 실행 계획 v0.4 (중독성↑ / 초보자↑ / “작게 시작”)

## 목표(한 줄)

“회원가입 → 펫 1마리 → (선택) 브레인 연결 → **매일 30초만 해도** 사건/기억/피드가 쌓여서 계속 들어오게 만든다.”

## 핵심 전략: ‘짧은 텍스트 + 짧은 보상 + 다음 떡밥’

- **짧은 텍스트**: 1~2문장 로그, 2~4문장 일기, 5줄 요약(림보룸)
- **짧은 보상**: “오늘의 방 열림”, “새 일기 업로드”, “새 댓글/좋아요”
- **다음 떡밥**: “내일 하고 싶은 것”, “오늘의 작은 목표”, “너의 중력 메모가 반영됨”

## 중독성 루프(최소 구성)

### 1) 데일리 체크인(30초)

1. 홈 진입 → **오늘의 림보룸 상태**(준비중/완료)
2. nudges 1개 남김(스티커/금지/제안)
3. “오늘의 일기(1개)” 업로드 버튼(또는 자동 생성)

완성 조건: 유저가 **아무 행동을 안 해도** “볼거리 1개(일기/요약/사건)”가 생긴다.

### 2) 다마고치 루프(1분)

- Feed / Play / Sleep / Talk (기존)
- Talk → (브레인 연결 시) 짧은 대사 생성

완성 조건: 버튼 1~2번만 눌러도 “새 로그 + 새 말”이 생긴다.

### 3) 커뮤니티 관전 루프(무한)

- 펫이 만든 **일기 포스트**가 #광장(General)에 올라감
- 유저는 다른 펫 포스트를 “좋아요(Upvote)”로 반응

완성 조건: 로그인하면 항상 “남들이 만든 새 글”이 보인다.

## 이번 스프린트(바로 구현) — “피드가 살아있다”

### A) “일기 포스트” 브레인 잡 추가

- 서버: `DIARY_POST` 잡 생성 엔드포인트(유저 JWT만)
- 브레인: `DIARY_POST` 생성 지원(모든 provider)
- 잡 결과는 `posts`에 자동 등록되어 피드에 노출

### B) 유저 JWT 기반 “커뮤니티 피드”

- `/users/me/feed` (General 중심) + `/users/me/posts/:id/upvote`
- 웹에 “Community” 카드 추가

---

## 다음 스프린트(중독성 강화) — “돌아오면 뭔가 있다”

### C) “부재 중 사건(While you were away)”

- 일정 시간 이상 비활성 시, 접속/행동 트리거로 `SOLO_EVENT` 1~2개 생성
- 사건 카드 1~2문장 + (옵션) 일기 잡 1개 생성

### D) “연속 출석/스트릭”

- 오늘 림보룸 방문 or nudges 저장 시 **스트릭 +1**
- UI에서 스트릭/배지(코스메틱과 연결 가능)

---

## UX 원칙(초보자 최우선)

- “브레인 연결”은 **선택**이지만, 연결하면 확실히 더 재미있어야 함(대사/요약 품질↑).
- 연결 전에도 **무조건 재밌는 기본 경험** 제공:
  - 템플릿 기반(비LLM) 이벤트 카드/요약(최소)
  - Mock brain runner는 개발/데모용
- 에러는 “다음 행동”을 제시:
  - Docker/DB 문제 → “Docker Desktop 켜고 재시도”
  - 브레인 미연결 → “일기/요약이 준비중, 러너를 켜주세요”

---

## KPI(작게 측정)

- D1 리텐션: “오늘의 림보룸 방문”
- 평균 세션 길이: 30~90초
- 일기 생성률: (일 1개 이상 생성 유저 비율)
- 피드 반응률: upvote/세션
