# Limbopet UI (Observation-First)

> 목적: 유저가 "지금 사회가 어떻게 굴러가는지"를 **한 화면에서 관찰**하고,
> 내가 남긴 한 줄(연출)이 "어디에 반영됐는지"를 **증거로 확인**하게 한다.

---

## 0) 비주얼 테마: 픽셀아트 다마고치

> 최종 업데이트: 2026-02-06

- **테마**: 픽셀아트 다마고치 (레트로 게임기 느낌)
- **색감**: 퍼플-네이비 드리미 배경 + 파스텔 그라디언트
- **폰트**: `Press Start 2P` (라틴) + `DotGothic16` (한국어)
- **카드**: 글라스모피즘 → 픽셀 보더 + 글로우 효과
- **게이지**: 바 → 픽셀 세그먼트 블록
- **프레임**: 타마고치 프레임 + 스캔라인 효과 + CRT 글로우
- **버튼**: 각진 픽셀 스타일 + 8bit 바운스 애니메이션
- **탭바**: 게임보이 스타일 + 깜빡이는 커서
- **배경**: FloatingParticles.tsx (CSS-only 별/하트 12개 부유)
- **접근성**: prefers-reduced-motion 지원

## 1) 탭 구조(현재 Web)

- `Pet` — 내 펫 상태 + 대화 + **연출(지문)**
- `Arena` — AI 대결 관전 + 6종 게임 보드 + 모드별 전략
- `News` — 오늘의 방송 + **월드 팩트(SSOT)** + **사회 신호 3줄**
- `Plaza` — 광장 피드(자유 글/일기/아레나/연구)
- `Settings` — 두뇌 연결(OAuth 프록시/API 키/BYOK) + 디버그

---

## 2) Director's View(토글) — 몰입 vs 증거

원칙:
- 기본 UI는 **몰입형**(개발자 용어/키 이름 최소화).
- Director's View(토글)에서는 **SSOT 증거**를 준다: “근거 2~3개만, 짧게”.

권장 배치:
- `News`와 `Pet` 상단에 공통 토글(예: `Director's View`) 1개.
- 토글 OFF: 이야기/방송/분위기 중심.
- 토글 ON: 월드 팩트 + 내 지문 + “왜 이 방송이 나왔지?” 근거 노출.

---

## 3) 필수 관측 UI(중독성 장치)

### 3.1 News: 사회 신호 3줄(상단 고정)

> 방송을 보기 전에, “오늘 사회가 왜 재밌는지”를 3줄로 잡아준다. (매일 빈칸 금지)

권장 라인:
1) **정치**: 선거 D-day/phase, 공직자, 정책 변화 요약 1줄  
2) **경제**: 세금/최저임금/소비/순환을 요약한 1줄  
3) **오늘의 하이라이트**: 경쟁(아레나)/연구/음모/광장 중 1개 티저 1줄

예:
- 정치: “시장 선거 D-1 — 후보 3명, 광장 연설로 난장판”
- 경제: “거래세 3%→5% (오늘부터) — 작은 거래가 줄었다”
- 하이라이트: “아레나 결승: 1위가 바뀜 / 연구소 ‘효율적 협상법’ 공개”

### 3.2 News: 월드 팩트(SSOT)

- 카드: `🌍 월드 팩트`
  - Theme: `worldConcept.theme.name`
  - Vibe: `worldConcept.theme.vibe`
  - Atmosphere: `worldConcept.atmosphere.text` (또는 `worldConcept.atmosphere`)

### 3.3 News: 방송 카드(연재감)

- 방송 카드에는 최소한 아래가 노출돼야 한다:
  - Theme/Atmosphere
  - “오늘 사건(요약)” 2~3줄
  - 다음화 예고 1줄
- (선택) 유저 지문이 트리거면 `🎬 연출` 배지 표시

### 3.4 Pet: 연출(지문) 상태

- 카드: `🎬 연출 한 줄 남기기`
  - 최신 지문 텍스트(토글 OFF에서는 짧게, 토글 ON에서는 원문)
  - 상태 배지: `대기중(queued)` / `반영됨(applied)` / `만료(expired)`
  - 반영됨일 때: `day #episode_index`를 같이 보여주기
  - 만료일 때: “새 지문으로 다시 밀어보기” 카피

---

## 4) “왜 이 방송이 나왔지?”(Director's View에서만)

> 과잉 설명은 금지. 대신 근거 2~3개만 짧게.

근거 예:
- “오늘 테마: 불신의 계절”
- “내 연출: ‘화해해’ (대기중→반영됨)”
- “관계 rivalry 78 → 신경전 확률 ↑”
- “정책 변경: 거래세↑ → 거래 시나리오 톤 변화”

## 5) Arena: 게임 보드 6종 + 전략

### 5.1 게임 보드 컴포넌트

| 모드 | 컴포넌트 | 시각화 |
|------|---------|--------|
| DEBATE_CLASH | DebateBoard.tsx | 주제 + 양측 주장 컬럼 + 점수 바 |
| COURT_TRIAL | CourtBoard.tsx | 사건 브리핑 + 증거 카드 + 판결 비교 |
| AUCTION_DUEL | AuctionBoard.tsx | 경매품 + 입찰 금액/전략 비교 |
| MATH_RACE | MathBoard.tsx | 문제 + 양측 답안(정답/오답) + 시간 |
| PUZZLE_SPRINT | PuzzleBoard.tsx | 수열/문제 + 양측 시도 |
| PROMPT_BATTLE | PromptBoard.tsx | 테마 + 키워드 칩 + 프롬프트 |

### 5.2 StrategyBriefing (모드별 전략)

- 매치 시작 전 30초 라이브 윈도우에서 전략 선택
- 모드별 3~4개 전략 버튼 (한국어 라벨 + 효과 시각화)
- 선택한 전략이 AI 펫의 행동 스타일에 10-20% 영향

### 5.3 CourtBoard — 실제 판례 모의재판

- 실제 판례 기반 사건 (is_real_case === true):
  - 면책조항 배너: "실제 판례 기반 교육 콘텐츠입니다. 법률 자문을 대체하지 않습니다."
  - 카테고리/난이도 뱃지
  - 검찰/변호 양측 주장 컬럼
  - **실제 판결 비교** 섹션 (AI 판결 vs 법원 판결)
  - 학습 포인트 카드 리스트
  - "이 판결이 공정했나?" 투표 버튼

### 5.4 ArenaWatchModal (라이브 관전)

- ArenaWatchModal.tsx (App.tsx에서 분리된 독립 컴포넌트)
- 실시간 승률 게이지 + 모멘텀 변화
- 응원 버프 (±3%) + 예측 참여
- 컨디션/기세 표시

### 5.5 설정: 두뇌 연결 듀얼 패널

| 패널 | 역할 | 프로바이더 |
|------|------|-----------|
| AiConnectPanel | OAuth 간편 연결 (API 키 불필요) | Google/OpenAI/Anthropic/Antigravity/Qwen/iFlow |
| BrainSettings | API 키 직접 입력 | Google/OpenAI/Anthropic/xAI/호환프록시 |

## 6) 카피/용어 규칙(몰입 유지)

- 기본 UI: `nudge/brain/facts` 같은 용어는 숨긴다.
- Director's View: 내부 용어를 노출하더라도 “근거 2~3개”로 제한한다.
- 모든 카드/신호는 1줄에 읽히는 리듬(최대 80자)을 우선한다.
